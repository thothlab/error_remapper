**Статус:** Выполнено  
**Результат:**
- Точное совпадение по коду (`key` == error code)
- Нечёткий поиск с комбинированным подходом:
  - Точное вхождение подстроки (case-insensitive) → скор 1.0
  - Скользящее окно + нормализованное расстояние Левенштейна

- Логика: 1 совпадение по коду → результат; несколько → нечёткий поиск среди них; 0 → нечёткий по всему словарю

## Задача 10: Настраиваемый формат ввода/вывода
**Статус:** Выполнено  
**Результат:**
- Вход: формат JSON полностью не зависит от структуры — поля для кода и сообщения извлекаются по настраиваемым именам (`statusCode`, `errorCode`, `code`, `errorText`, `message`, `title` и т.д.)
- Выход: формат описывается шаблоном в `[output.template]` секции `settings.toml`
- Доступные плейсхолдеры в шаблоне:
  - `{{code}}` — ремапленный код ошибки
  - `{{description}}` — ремапленное описание
  - `{{matched}}` — найдено ли совпадение (true/false)
  - `{{original_code}}` — исходный код из входного JSON
  - `{{original_message}}` — исходное сообщение из входного JSON
  - `{{input.FIELD}}` — любое поле из входного JSON (поддержка вложенности: `input.error.detail`)
- Добавлен CLI-флаг `--pretty` для форматированного вывода
- `RemapResult` расширен полями `original_code`, `original_message`
- 4 новых unit-теста в `output.rs` (простые плейсхолдеры, passthrough полей, вложенный доступ, boolean)
- Обновлён интеграционный тест `test_no_match_returns_original`
- Всего тестов: **28** (19 unit + 9 integration), все проходят

**Пример:**  
Вход: `{"statusCode": "3011", "errorText": "Не пройден фрод", "ErrorDescription": "Процесс не был пройден через антифрод"}`  
Выход: `{"statusCode": "81005", "errorText": "Перевод отклонён банком получателя", "ErrorDescription": "Процесс не был пройден через антифрод"}`

### Ключевые решения
- **Алгоритм нечёткого поиска**: комбинация точного вхождения подстроки и скользящего окна с нормализованным Левенштейном. Быстрый (без внешних NLP-библиотек), корректно работает с кириллицей.
- **Гибкий парсинг JSON**: рекурсивный поиск полей — поддерживает любой уровень вложенности.
- **Настраиваемость**: имена полей, порог нечёткости, пути к файлам, формат вывода — всё через TOML.
- **Шаблонный вывод**: формат выходного JSON полностью описывается в настройках с поддержкой passthrough полей из входного JSON.
