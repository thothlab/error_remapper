Требуется разработать алгоритм + консольную утилиту на rust для поиска ошибок, которые надо заменить на кастомные.

Основной конфиг с ошибками - это yanl файл, примерно такого формата:

preprocess-error:
  vocabulary:
    - key: "2001"
      substring: "unexpected symbol:"
      customDesc: "Недопустимый символ %s в назначении перевода"
      code: "81002"
    - key: "2002"
      substring: "Уточните у получателя"
      code: "81001"
    - key: "3010"
      substring: "Мы заблокировали переводы"
      code: "81003"
    - key: "3011"
      substring: "Система быстрых платежей заблокировала"
      code: "81004"
    - key: "3011"
      substring: "Не пройден фрод"
      customDesc: "Перевод отклонён банком получателя"
      code: "81005"
    - key: "3012"
      substring: "отклонен провайдером"
      customDesc: "Перевод отклонён банком получателя"
      code: "81006"
      
      
    Со смешной системы приходить json с ошибкой, в которой есть код ошибки и текст ошибки.   Например:
    	{"error":{"code":-2,"title":"Сервер временно недоступен. Уже работаем над этим, попробуйте позже","uuid":"ddaa650d-5cac-4ba4-8378-6745bf3b5b9d","details":"От ПЦ ЭК не получен ответ с HTTP статусом 200 EWallet CircuitBreaker error: CircuitBreaker 'pcekCircuitBreaker' is OPEN and does not permit further calls"}}
    	
    Формат может быть иной. Главное есть "code" или "errorCode"
    и "title" (может быть "message", "errorMessage") - список полей можно задать в файле с настройками утилиты.
    
    если есть "code" или "errorCode" то по ним происходит поиск ошибок описанных в yaml файле. Если есть однозначное совпадение - найдена одна запись у которой код совпадает с "key" - утилита (алгоритм) выдает json: {code:"XXXX", customDesc:"YYYYYY"} - это если есть в yaml customDesc
    Если  customDesc нет - выдается :  {code:"XXXX", customDesc:"то что пришло в title изначально"} 
    
    Если точного совпадения по code и key нет (может не вернуться ни одной записи по code либо записей будет несколько) - алгоритм производит неточный поиск "substring" из yaml с текстом title из ответа смежной системы.
    Если записей нет - ищет неточно по всему yaml.
    Если записей вернулось несколько - то только по ним.
    
    Наиболее подходящую строку из yaml оборачивает в формат 
    {code:"XXXX", customDesc:"ZZZZZZ"}
    
    ZZZZZZ -  это либо customDesc если есть в найденой записи в yaml, либо то что пришло в title изначально
    
    Утилита должна быть на rust, основной алгоритм тоже.
    Алгоритм нечеткого поиска подбери максимально точный и быстрый для такой задачи.
    Тексты ошибок могут быть на русском (в основном), но и на английском (реже), если это важно.
